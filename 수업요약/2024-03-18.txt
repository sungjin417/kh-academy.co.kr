# python -> 제일 깔끔한 언어
//특징
1. 간결하다(인터프리터 언어이기 때문에 1줄만 있어도 된다)
2. ' ; ' 이 없고 들여쓰기를 해줘야함
3. 인터프리터(Interpreter)언어로 프로그램 코드가 한 라인 씩
순서대로 해석되고 실행 되지만, 컴파일 언어에 비해 느리다(60~70배정도).
4. 멀티 패러다임 언어로 절차지향언어와 객체지향언어의 특성을 모두 가지고 있다.
5. 다양한 라이브러리가 풍부하고, 다른 언어와 연동성이 좋음

//단점
1. 시스템과 밀접한 프로그래밍 영역(느리기때문에)
ex.) 운영체제, 대규모의 연산이 필요한 영상처리, 압축알고리즘등
2. 임베디드 시스템(하드웨어와 밀접한 관련이 있는 분야
3. 모바일 프로그래밍(App)

파이썬 입출력 함수
대괄호 [] : 리스트를 표시할때 사용 
중괄호 {} : 딕셔너리(자바의 맵과 비슷)사용
소괄호 () : 함수의 인수, 튜플
